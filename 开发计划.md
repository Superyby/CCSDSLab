# CCSDS 协议仿真与测试平台 V1.0
## 1. 后端核心功能规划
我们将后端划分为 4 个核心模块，每个模块独立开发、独立测试，最后组装成完整的仿真引擎。
### 1.1 模块一：协议解析引擎 (Protocol Parser)
目标：能够准确解析 CCSDS TM/TC 帧的二进制数据。
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| TM 帧解析 | 解析遥测帧头（Version, Type, SCID, VCID, Length, CRC） | 🔴 高 |
| TC 帧解析 | 解析遥控帧头（支持分段包重组） | 🔴 高 |
| CRC-16-CCITT 校验 | 实现标准的 CRC 算法，验证帧完整性 | 🔴 高 |
| 位域操作工具类 | 封装位运算，简化帧头解析代码 | 🟡 中 |
| 错误检测与报告 | 识别并报告帧格式错误（长度不匹配、CRC 错误等） | 🟡 中 |

### 1.2 模块二：数据仿真生成器 (Data Simulator)
目标：模拟卫星或地面站发送的真实数据流。
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 随机 TM 帧生成 | 生成符合 CCSDS 标准的随机遥测帧（含有效载荷） | 🔴 高 |
| 固定模式帧生成 | 生成特定 SCID/VCID 的测试帧（用于单元测试） | 🟡 中 |
| 注入错误帧 | 故意生成 CRC 错误或长度错误的帧，测试解析器鲁棒性 | 🟢 低 |
| 时间戳标记 | 为生成的帧添加模拟时间戳 | 🟢 低 |

### 1.3 模块三：通信接口层 (Communication Interface)
目标：提供数据收发能力（先本地，后扩展网络）。
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 本地内存队列 | 使用 `std::queue` 或 `boost::lockfree::queue` 实现线程安全的数据缓冲 | 🔴 高 |
| 文件读写接口 | 从二进制文件读取帧数据，或将解析结果写入文件 | 🟡 中 |
| TCP/UDP 服务端 | 监听端口，接收外部仿真器发送的帧数据 | 🟢 低 (V1.1) |
| 串口模拟 | 模拟串口通信（可选） | 🟢 低 |

### 1.4 模块四：测试与日志系统 (Testing & Logging)
目标：确保代码质量，方便调试。
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| Google Test 集成 | 编写单元测试，验证解析器正确性 | 🔴 高 |
| spdlog 日志系统 | 输出调试信息、错误报告、性能统计 | 🟡 中 |
| 性能基准测试 | 测试解析器的吞吐量（帧/秒） | 🟢 低 |

## 2. 开发计划
### 2.1 第一阶段 基础框架搭建 
1. 完善目录结构
```text
CCSDSLab/
├── include/ccsds/
│   ├── frame.hpp          # 帧结构定义
│   ├── parser.hpp         # 解析器接口
│   ├── simulator.hpp      # 仿真器接口
│   └── crc.hpp            # CRC 工具
├── src/
│   ├── frame/
│   │   ├── tm_frame.cpp
│   │   └── tc_frame.cpp
│   ├── parser/
│   │   └── frame_parser.cpp
│   ├── simulator/
│   │   └── frame_generator.cpp
│   └── utils/
│       └── crc16.cpp
├── test/
│   ├── test_parser.cpp
│   └── test_simulator.cpp
├── CMakeLists.txt
└── main.cpp
```
2. 实现 CRC-16-CCITT 算法 (src/utils/crc16.cpp)
```c++
uint16_t crc16_ccitt(const uint8_t* data, size_t length);
```
3. 完成 TMFrame 解析类 (src/frame/tm_frame.cpp)
- 构造函数解析二进制数据
- isValid() 检查 CRC 和长度
- getHeader(), getPayload() 访问器
### 2.2 第二阶段 核心功能实现
1. 实现 FrameParser 类
    - 自动识别 TM/TC 帧
    - 返回解析后的对象或错误码
2. 实现 FrameGenerator 类
    - 生成随机/固定模式的 TM 帧
    - 支持配置 SCID, VCID, 数据长度
3. 集成 Google Test
    - 编写测试用例：正常帧、错误帧、边界情况

### 2.3 第三阶段 集成与优化
1. 主程序整合
    - main.cpp 中创建解析器和仿真器
    - 循环生成帧 → 解析帧 → 输出结果
2. 添加日志系统
    - 使用 spdlog 输出解析过程
    - 统计成功/失败帧数量
3. 性能测试
    - 测试每秒能处理多少帧
    - 优化内存分配（使用对象池）
